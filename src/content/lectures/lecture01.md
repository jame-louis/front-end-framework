## 一、Web的诞生与发展

### 1.1 Web的起源 (1989-1995)

- **1989年** Tim Berners-Lee 在CERN提出WWW构想
- **1991年** 第一个网站 http://info.cern.ch 上线
- **1993年** CERN宣布Web技术免费开放
- **核心技术三剑客**：HTML + HTTP + URL
- **目的**：方便科学家之间分享研究文献

### 1.2 Web vs Internet

**Internet（互联网）**
- 全球计算机网络基础设施
- 发明于1969年（ARPANET）
- 像公路系统
- 包含Email、FTP、Web等多种服务

**WWW（万维网）**
- 运行在互联网上的信息服务
- 发明于1989-1991年
- 像公路上跑的汽车
- 仅是Internet的一种应用

## 二、Web的三个时代

### 2.1 Web 1.0：静态门户时代 (1990-2004)

- **核心特征**：只读（Read-only）
- **用户角色**：内容消费者
- **数据归属**：网站拥有
- **代表产品**：Yahoo!门户、新浪、搜狐、网易
- **技术栈**：HTML 4.0、CSS 1/2、JavaScript (ES3)

### 2.2 Web 2.0：社交互动时代 (2004-至今)

- **核心特征**：读+写（Read-write）
- **用户角色**：内容创造者
- **数据归属**：平台拥有
- **代表产品**：Facebook、微信、抖音、B站、淘宝
- **问题**：平台垄断、数据孤岛、隐私侵犯

### 2.3 Web 3.0：去中心化时代（正在兴起）

- **核心特征**：读+写+拥有（Read-write-own）
- **用户角色**：内容拥有者+治理参与者
- **数据归属**：用户自己拥有（区块链存储）
- **技术基础**：区块链、智能合约、代币/通证
- **当前挑战**：用户体验差、性能瓶颈、监管不确定

### 2.4 Web三个时代对比

| 特性 | Web 1.0 (1990-2004) | Web 2.0 (2004-至今) | Web 3.0 (兴起中) |
|------|---------------------|---------------------|------------------|
| 交互模式 | 只读 | 读写 | 读写+拥有 |
| 用户角色 | 消费者 | 创造者 | 所有者 |
| 数据归属 | 网站管数据 | 平台管数据 | 自己管数据 |
| 商业模式 | 门户广告 | 平台经济 | 代币经济 |
| 核心技术 | HTML/CSS | Ajax/移动 | 区块链 |

## 三、前端技术演进

### 3.1 第一阶段：静态页面时代 (1990-2004)

- **特征**：网页以展示静态内容为主，前后端混合开发
- **技术栈**：HTML 4.0、CSS 1/2、JavaScript (ES3)
- **服务端技术**：ASP、JSP、PHP 渲染完整页面
- **JavaScript用途**：仅用于简单表单验证和页面特效

### 3.2 第二阶段：动态交互时代 (2005-2012)

- **2005年** Google Maps 发布，Ajax 技术开始流行
- **2006年** jQuery 发布，极大简化了 DOM 操作
- **关键技术**：Ajax 无刷新交互、JSON 取代 XML
- **发展趋势**：JavaScript 开始承担更多职责，前后端分离萌芽

### 3.3 第三阶段：工程化时代 (2013-至今)

- **2013年** React 发布，引入虚拟 DOM 和组件化
- **2014年** Vue.js 发布，2015年 ES6 发布
- **核心特征**：组件化开发、单页应用 (SPA)
- **前端工程化**：构建工具、模块化、自动化测试

## 四、前后端架构对比

### 4.1 传统 B/S 架构（服务端渲染 SSR）

```
浏览器          Web服务器          数据库
  |                |                |
  |--- 请求页面 --->|                |
  |                |--- 查询数据 -->|
  |                |<-- 返回数据 ---|
  |                |                |
  |                |  渲染完整HTML  |
  |<-- 返回页面 ----|                |
```

**特点**：
- 服务端负责数据查询和页面渲染
- 浏览器只负责展示HTML
- 每次交互需刷新页面

### 4.2 现代 B/S 架构（前后端分离）

```
浏览器（前端应用）      API服务器         数据库
  |                   |               |
  |--- 请求静态资源 ->|               |
  |<- 返回JS/CSS/HTML-|               |
  |--- API请求(JSON)->|               |
  |<- 返回JSON数据 ---|               |
  |   前端渲染页面                      |
```

**特点**：
- 前端独立部署，通过AJAX/Fetch与后端通信
- 数据交互使用JSON格式
- 用户体验接近原生应用

## 五、Node.js 与前端工程化

### 5.1 Node.js 的革命性影响

> "Node.js让JavaScript突破了浏览器的边界，开启了全栈开发的新纪元。"
> — Ryan Dahl, 2009

### 5.2 Node.js 的核心贡献

- **统一编程语言**：前后端使用同一种语言 JavaScript
- **npm 生态**：全球最大的开源包管理器，200万+包
- **前端工程化工具链**：Webpack、Vite、Babel
- **全栈开发**：Express/Nest.js + Vue/React

## 十、基础回顾

### 10.1 HTML5 语义化标签

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>语义化页面</title>
</head>
<body>
    <header>
        <nav>导航链接</nav>
    </header>
    <main>
        <article>
            <section>内容区块</section>
        </article>
        <aside>侧边栏</aside>
    </main>
    <footer>版权信息</footer>
</body>
</html>
```

**语义化标签的优势**：
- 提升代码可读性和可维护性
- 有利于 SEO 搜索引擎优化
- 便于屏幕阅读器解析，提升无障碍访问
- 常用标签：`header`、`nav`、`main`、`article`、`section`、`aside`、`footer`

### 10.2 Flexbox 核心属性

```css
/* 容器属性 */
.container {
    display: flex;
    flex-direction: row;        /* 主轴方向 */
    justify-content: center;    /* 主轴对齐 */
    align-items: center;        /* 交叉轴对齐 */
    flex-wrap: wrap;            /* 是否换行 */
    gap: 20px;                  /* 项目间距 */
}

/* 项目属性 */
.item {
    flex: 1 1 200px;            /* grow shrink basis */
    align-self: flex-start;     /* 单独对齐 */
}
```

### 10.3 Grid 布局示例

```css
.grid-container {
    display: grid;
    grid-template-columns: 200px 1fr 200px;
    grid-template-rows: auto 200px;
    gap: 20px;
}

.header { grid-column: 1 / -1; }   /* 跨越所有列 */
.sidebar { grid-column: 1; }
.main { grid-column: 2; }
.aside { grid-column: 3; }
.footer { grid-column: 1 / -1; }
```

### 10.4 响应式设计：媒体查询

```css
/* 移动优先：基础样式 */
.box {
    width: 100%;
    padding: 15px;
}

/* 平板设备（≥768px） */
@media screen and (min-width: 768px) {
    .box { width: 48%; }
}

/* 桌面设备（≥1024px） */
@media screen and (min-width: 1024px) {
    .box { width: 30%; }
}
```
